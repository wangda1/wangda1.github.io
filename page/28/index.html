花指令


这里以看雪的CTF2017秋季赛第二题为例

  • 首先关于第一个迷惑方程组的出现，需要scanf的栈溢出绕过
  • 因为字符限制0-9,a-z,A-Z不存在多解，需要找到另一部分验证函数
  • 验证函数部分去除花指令
  • 关于花指令的去除
  • 观察特征：出现大量的跳转，并且在IDA，OD中均不能反汇编
  • OD加载：开启OD的 trace into ，debug的单步步入跟踪

  • 下断点：在关键跳转没有实现的地方下断点 ，运行到此处修改寄存器标志位帮助跳转实现，trace前面没有跟到的验证代码

编码与解码

  • base64： 组成：a-z,A-Z,+,-,=共65个字符，对字符进行 utf8,gb2312,unicode编码后以4字节替换3字节方式进行编码，最终字节数目会变成原来的4/3左右。

  • 特征1：以1个字节结尾的，编码后结尾会出现”==”

  • 特征2：以2个字节结尾的，编码后结尾会出现”=”

  • Crypto模块：

  • 关于AES加密：AES属于块加密，加密的几个主要参数有：

  • key:秘钥（根据秘钥长度有AES128,AES256）

  • mode：加密模式 （ECB,CBC,CFB等）

  • padding：对最后一块的填充（PKCS5,PKCS7,NOPADDING）

  • iv:作用与key类似，称为初始向量

  • AES加密： from Crypto.Cipher import AES obj = AES.new(key,mode,iv) obj.encrypt( ) obj.decrypt( )

  • hash散列：from Crypto.Hash import SHA256 hash = SHA256.new( ) hash

8086标志寄存器

发表于 2018-04-19 | 分类于 CTF ， RE

关于8086标志寄存器flag

16位寄存器存储的信息通常被称为 程序状态字(PSW)

  • ZF: zero flag针对于运算指令，为零则为1

** ** JE/JZ,JNE/JNZ指令与此

  • PF: parity flag: 0–odd,1–even
  • SF: sign flag: 针对于有符号数运算结果 1–为负，0–为非负（判断标准为最高位是否为1）
  • CF: cary flag: 进位标志：针对于无符号数运算
  • OF: overflow flag: 溢出标志： 针对于有符号数运算

这些标志位在运算的时会发生改变，要根据是否为有符号数运算来判断是否有作用

x86汇编指令

发表于 2018-04-19 | 分类于 CTF ， RE

x86汇编指令

  • retn,retf：返回指令 retn近返回，retf 远返回；相当于：pop eip; pop eip,pop cs

  • retn 操作数：pop结束之后 sp 加上操作数

  • pusha,popa;pushad,popad: pusha,popa对应于8个 通用寄存器：AX（累计器）,BX（基址寄存器）,CX（计数寄存器）,DX（数据寄存器）,BP（基址指针寄存器）,SP（堆栈指针寄存器）,SI（源地址寄存器）,DI（目的地址寄存器）；它们的入栈顺序为；AX,CX,DX,BX,SP(初始值)，BP,SI,DI


  • 跳转指令：
  • jmp short 标号；jmp near ptr 标号；jcxz 标号；loop 标号 都是相对位置转移；
  • jmp far ptr 标号；jmp 16 reg; jmp word ptr 内存单元地址（段内转移）；jmp dword ptr 内存单元地址（段间转移）都是绝对位置转移


  • mul指令

  • 默认一个参数存储在AL/AX中，另一个参数为reg/内存单元；当结果为16位时存储在AX中，当结果为32位时，高16位存储在DX,低16位存储在AX中

  • jnz,jne: jmp当ZF=0即比较两者不相等的时候

  • jz,je: jmp当ZF=1即两者相等的时候
