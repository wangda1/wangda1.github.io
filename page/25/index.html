《程序员的自我修养》阅读笔记

装载过程：（静态）

  • 操作系统读取可执行文件的头部
  • 检查文件的合法性
  • 将头部的header的segment虚拟地址，文件地址，属性，并映射至PVM(Process virtual memory)
  • 可执行文件的入口进行一些初始化操作

装载过程：（动态）

  • 前述操作与上述相似
  • 映射完毕可执行文件后，启动动态链接器(Dynamic Linker)
  • ld.so对可执行文件进行动态链接工作
  • 控制权转交至可执行文件入口

系统调用

发表于 2018-04-25 | 分类于 CTF ， PWN

系统调用

32位模式下



linux是通过int 80h来实现系统调用的

  • 系统调用号存入 eax 寄存器，通过 int 80h开启中断，启动系统调用
  • 参数存入的寄存器：EAX,EBX,ECX,EDX,ESI,EDI
  • 参数的返回寄存器：EAX


linux系统调用的一般流程：

  • 应用程序调用库函数API
  • API将调用号传入EAX，并且启动访管中断陷入内核态
  • 内核中断处理函数调用对应的内核函数（真系统调用）
  • 内核系统调用函数将返回值传入EAX返回到中断处理函数
  • 中断处理函数返回到API中


关于一般系统调用的寄存器（除EBP,ESP的通用寄存器）相关设置查linux系统调用表

系统调用号位于 unistd32/64.h内，可通过 locate unistd32.h查看


64位模式下

  • 关于系统调用号仍然是存放在 rax 寄存器中，但与 32位下的系统调用号不同
  • 用户模式下的系统调用传递参数的寄存器有：rdi,rsi,rdx,rcx,r8,r9
  • 内核模式下的系统调用传递参数的寄存器有：rdi,rsi,rdx,r10,r8,r9
  • 通过 syscall 指令启用异于 32位模式下的 int 80h
  • 返回值同样存入 rax 寄存器用于返回

基本ROP

  • ret2text: 针对于存在system(“/bin/sh”)
  • 寻找到system(“/bin/sh”)的函数入口；
  • 溢出到函数入口处拿shell


  • ret2shell: 针对于存在可执行段并且可读入指令
  • 存在可执行段如：.bss段
  • 存在输入及复制函数
  • 输入shellcode，并溢出至可执行段(.bss)拿shell


  • ** **ret2sys: 这个针对面较广
  • 条件：多个带有pop的gadget，int 80h（linux）
  • 找多个gadget来准备syscall条件
  • 准备数据满足gadget构造payload
  • 工具：ROPgadget找gadget的好工具

花指令

发表于 2018-04-24 | 分类于 CTF ， RE

花指令


这里以看雪的CTF2017秋季赛第二题为例

  • 首先关于第一个迷惑方程组的出现，需要scanf的栈溢出绕过
  • 因为字符限制0-9,a-z,A-Z不存在多解，需要找到另一部分验证函数
  • 验证函数部分去除花指令
  • 关于花指令的去除
  • 观察特征：出现大量的跳转，并且在IDA，OD中均不能反汇编
  • OD加载：开启OD的 trace into ，debug的单步步入跟踪
  • 提取有效指令：一般除了跳转，当然关键跳转不能略过（关键跳转一般跟在sub，cmp等之后因为需要判断往往是jz,jnz等之类的唯一性跳转）
  • 下断点：在关键跳转没有实现的地方下断点 ，运行到此处修改寄存器标志位帮助跳转实现，trace前面没有跟到的验证代码

编码与解码

  • base64： 组成：a-z,A-Z,+,-,=共65个字符，对字符进行 utf8,gb2312,unicode编码后以4字节替换3字节方式进行编码，最终字节数目会变成原来的4/3左右。

  • 特征1：以1个字节结尾的，编码后结尾会出现”==”

  • 特征2：以2个字节结尾的，编码后结尾会出现”=”

  • Crypto模块：

  • 关于AES加密：AES属于块加密，加密的几个主要参数有：

  • key:秘钥（根据秘钥长度有AES128,AES256）

  • mode：加密模式 （ECB,CBC,CFB等）

  • padding：对最后一块的填充（PKCS5,PKCS7,NOPADDING）

  • iv:作用与key类似，称为初始向量

  • AES加密： from Crypto.Cipher import AES obj = AES.new(key,mode,iv) obj.encrypt( ) obj.decrypt( )

  • hash散列：from Crypto.Hash import SHA256 hash = SHA256.new( ) hash


  • 进制与字符之间的转换
  • 16进制转ASCII: ‘hex_number’.decode(‘hex’)

str.encode(‘hex’)

  • chr(),ord()函数
  • chr()将数转换成Ascii码
  • ord()将Ascii码转换成数
